# fledge-support

Public support hub for [Fledge Practice](https://fledgepractice.com), the practice-management tool for speech-language pathologists. Lives at <https://support.fledgepractice.com>.

Content-only, no auth, no PHI. The product itself (which is HIPAA-covered) lives in a separate repo — see [Related repos](#related-repos).

## Stack

- [Astro](https://astro.build) with the [Starlight](https://starlight.astro.build) docs theme — search, sidebar nav, article TOC, MDX, dark mode, accessibility for free.
- Tailwind / Starlight CSS variable overrides in `src/styles/custom.css` to match the Fledge palette (cream background, forest-teal accent, terracotta clay accent, Crimson Pro display serif).
- [Pagefind](https://pagefind.app) for static client-side search, indexed at build time.
- Hosted on [Vercel](https://vercel.com) — production branch deploys on push to `main`, PR previews automatic.
- Node 22.12+ (or any Node 22 LTS that ships with Astro 6 support). pnpm.

## Local dev

```bash
pnpm install
pnpm dev          # http://localhost:4321
pnpm build        # static build to ./dist
pnpm preview      # serve ./dist locally
pnpm exec astro check   # frontmatter + MDX validation
```

## Content authoring

Articles are MDX files under `src/content/docs/<category>/<slug>.mdx`. Six top-level categories exist; each has its own folder and is autogenerated into the sidebar by `astro.config.mjs`.

```
src/content/docs/
  index.mdx                 # homepage (splash with hero + tile grid)
  getting-started/
  clients/
  sessions-notes/
  invoices-payments/
  account-billing/
  privacy-security/
```

Each article needs frontmatter:

```yaml
---
title: Adding a client
description: How to add a new client to your Fledge roster.
sidebar:
  order: 1
lastUpdated: 2026-04-29
---
```

Order within a category is set by `sidebar.order` (lower comes first). The `description` powers Pagefind result snippets — write it for someone scanning a search list.

## Authoring rules

This is a public site backed by a HIPAA-covered product. Two hard rules:

1. **No PHI placeholder content.** Never put fake client names, fake dates of birth, fake session notes, or any other identifying detail into an article — even as an example. If you need to demonstrate a workflow, use generic placeholders ("Open the client's profile") and screenshots that show only your own test data.
2. **No off-domain trackers.** The site has no analytics today. If we add any, it'll be a self-hosted Plausible instance or equivalent. Do not add Google Analytics, Meta Pixel, Hotjar, etc. — adding tracking on a support site that providers visit while logged in to a HIPAA-covered app is a privacy issue we don't want.

## Deploy

Pushes to `main` auto-deploy to <https://support.fledgepractice.com> via Vercel. Pull requests get a preview URL.

The custom domain is configured in Vercel; the Route 53 record (`support.fledgepractice.com` → `cname.vercel-dns.com`) lives in the AWS account that owns the rest of the Fledge infrastructure.
